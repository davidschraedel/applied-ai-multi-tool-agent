LangChain Overview
==================

LangChain is an open-source framework for building applications powered by large language models (LLMs). It provides abstractions for common LLM tasks including prompt management, chaining, memory, agents, and retrieval-augmented generation (RAG).

Core Concepts
--------------
- **Chains**: Sequences of calls to LLMs or utilities. The most basic chain takes a prompt template and an LLM and produces a string output.
- **Agents**: LLMs that can choose and use tools dynamically based on user input. The ReAct pattern is the most common: Reason then Act.
- **Memory**: Mechanisms to retain information across conversation turns. LangChain supports in-memory buffers, vector store retrievers, and external databases.
- **Retrievers**: Components that fetch relevant documents for a query. Used in RAG pipelines.
- **Tools**: Functions an agent can call. Each tool has a name, description, and input schema (usually defined with Zod in LangChain.js).

LangChain.js vs Python
-----------------------
LangChain has both a Python and a JavaScript/TypeScript (LangChain.js) library. The APIs are similar but not identical. LangChain.js uses ESM modules in v1.x, and many integrations now live in separate packages like @langchain/openai, @langchain/anthropic, and @langchain/classic.

Why Use LangChain?
-------------------
LangChain handles the plumbing: message formatting, tool binding, retry logic, and streaming. This lets developers focus on application logic rather than LLM API details. It also provides a consistent interface across different LLM providers (OpenAI, Anthropic, Google, etc.).

Version Notes
--------------
LangChain.js v1.x (2025) reorganized packages significantly. Core types live in @langchain/core. Vector stores and legacy document loaders moved to @langchain/classic. The AgentExecutor is deprecated in favor of LangGraph's createReactAgent.
