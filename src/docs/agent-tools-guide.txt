AI Agent Tools — Calculator, Web Search, and Document Retrieval
================================================================

This document describes the three tools available in this AI agent, their purposes, and how to use them effectively.

Calculator Tool
---------------
Purpose: Evaluates mathematical expressions and returns the result.
When to use: Any time the user asks to compute, calculate, or evaluate a mathematical expression.
How it works: The expression is sanitized (only arithmetic operators allowed), then evaluated using the Function constructor. Returns an error string if the expression is invalid.

Examples of valid inputs:
  - "2 + 2" → "2 + 2 = 4"
  - "42 * 17" → "42 * 17 = 714"
  - "(100 - 32) * 5/9" → converts Fahrenheit to Celsius
  - "2 ** 10" → "2 ** 10 = 1024"

Error handling: Invalid expressions (e.g., "hello world") return a descriptive error string instead of throwing an exception. The agent can then inform the user or retry with a corrected expression.

Web Search Tool
----------------
Purpose: Searches the internet for current or factual information using the Tavily API.
When to use: Questions about recent events, current facts, sports results, stock prices, news, or anything that requires up-to-date information the LLM may not have in its training data.
How it works: Sends the query to Tavily, which performs a web search and returns a synthesized answer with sources. Returns the top 3 results.

Best practices:
  - Be specific in queries for better results (e.g., "latest iPhone model 2025" not just "iPhone")
  - Use for facts that change over time

Document Retrieval Tool (RAG)
------------------------------
Purpose: Searches internal documentation for project-specific or domain-specific information.
When to use: Questions about LangChain, LangGraph, RAG, logging, or any topic covered in the loaded documents.
How it works: Embeds the query using OpenAI's text-embedding-3-small model, performs similarity search in the in-memory vector store, and returns the top 3 most relevant document chunks with source attribution.

Return format:
  [Source: src/docs/langchain-overview.txt]
  <relevant text chunk>

Always cite the source so the user knows where the information came from.

Tool Selection
--------------
The agent automatically selects the right tool based on the user's question and each tool's description. Tool descriptions are the primary signal the LLM uses to choose — they should be specific and include examples of the kinds of questions each tool handles.
